=============================================================
1. SCOPO DEL SISTEMA
=============================================================

LOGOS è un motore dati universale basato su un modello
EVENT LEDGER.

Il sistema registra eventi operativi ed economici
e li trasforma in dataset analitici.

Il motore è progettato per essere:

- universale
- replicabile
- leggero
- indipendente dal progetto

Le istanze previste:

LOGOS_ADEXIMA
LOGOS_ASPRI
LOGOS_MAURIZIOLAB
LOGOS_AI0S (potenziale)


=============================================================
2. PRINCIPIO ARCHITETTURALE
=============================================================

LOGOS utilizza il paradigma:

EVENT SOURCING

cioè:

il database non contiene stati aggregati
ma registra eventi elementari.

Schema logico:

INPUT
↓
EVENT LEDGER
↓
DATASET ANALITICI
↓
DASHBOARD

=============================================================
11. DATASET ANALITICI
=============================================================

Questi fogli trasformano il ledger in informazioni.

Fogli:

CONTROLLO_PROJECT
CONTROLLO_PROJECT_DETTAGLIO
CONTROLLO_ENTITY
CONTROLLO_ENTITY_DETTAGLIO


Funzioni:

aggregazione eventi
calcolo ore
calcolo costi
calcolo incassi
calcolo margini


=============================================================
12. FORMULE ANALITICHE
=============================================================

Le aggregazioni usano:

ARRAYFORMULA
MAP
SUMIFS
COUNTIFS

Gli intervalli sono limitati per performance:

RAW_DATA → 2:2000
PROJECTS → 2:200
ENTITIES → 2:500
